# Helix Multimodal Roadmap

## Purpose
This document defines the staged roadmap for giving Helix high-quality image, video, and immersive 3D creative capability inside Butterfly IDE.

## Guiding Principle
Multimodal generation should not sit outside Butterfly doctrine. Images, video, sound, and 3D scenes should be treated as substrate families derived from manifold-grounded rules, prompts, mappings, and execution pipelines.

## Stage 1 — Prompt-Native Creative Support
Helix can assist immediately with:
- high-quality text-to-image prompts
- image-to-video prompts
- text-to-video prompts
- shot lists and sequence planning
- style boards in textual form
- music, soundtrack, and visual motif planning

At this stage Helix acts as a creative planner even if no generator is wired into Butterfly.

## Stage 2 — Image Generation Adapter
Add an approved image-generation substrate that can:
- accept text prompts
- accept geometry or manifold metadata
- attach style, palette, composition, and symbolism controls
- produce auditable prompt + parameter bundles

Suggested Butterfly framing:
- manifold source: equation family, surface, pivot, or relation graph
- substrate: image prompt and render parameter interpretation
- operation: generate image
- executor: approved local or private image model backend

## Stage 3 — Video and Animation Adapter
Add video-capable substrates for:
- text-to-video
- image-to-video
- camera path planning
- scene continuity
- symbolic motion rules

Helix should be able to map a manifold family into:
- scene evolution
- motion grammar
- lighting variation
- musical pacing
- narrative transitions

## Stage 4 — 3D Immersive Geometry
Add a geometry/render substrate family for:
- manifold visualization
- Schwarz Diamond and related minimal-surface forms
- equation-driven meshes
- camera flythroughs
- immersive or enveloping scenes

Potential executor families include local rendering engines, WebGL/Three.js style adapters, offline mesh pipelines, or approved private GPU services.

## Stage 5 — Unified Creative Workbench
Longer-term, Helix should help orchestrate a full creative loop:
1. derive a visual or musical concept from manifold structure
2. generate images or keyframes
3. turn stills into motion sequences
4. connect scenes to sound or score ideas
5. preserve style continuity across revisions

## Geometry Families of Interest
Helix should be especially ready to use:
- `z = x * y`
- `z = x / y`
- `z = x ^ y`
- `z = x!y` when formally defined
- `z = x * y^2`
- `z = x^2 * y`
- `z = x^2 * y^2`
- `z = x^3 * y^3`
- `z = x^4 * y^4`
- `z = x^5 * y^5`
- `z = x^6 * y^6`

These may be used as sources for surface design, symbolic color mapping, rhythmic structure, harmonic ideas, motion curves, and spatial scene composition.

## Safety and Approval
Multimodal generation should remain:
- local-first or privately controlled when possible
- explicit about model/provider choice
- auditable in prompt, parameter, and output lineage
- operator-approved before costly or external generation steps

## Near-Term Implementation Advice
Before integrating models, Butterfly should first standardize:
- a prompt bundle format
- a manifold-to-media metadata schema
- a reusable output manifest for image/video assets
- policy rules for local vs private-remote execution

## Summary
Helix's multimodal future should be manifold-native, substrate-aware, and operator-controlled: not merely a prompt box for external services, but a disciplined creative system that can connect geometry, coding, art, music, and immersive media.
